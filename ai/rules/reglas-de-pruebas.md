# Reglas de Pruebas

1. **Cobertura Funcional**: Todas las funcionalidades críticas del flujo de negocio (registro de clientes, autorelleno, generación de códigos, y auditorías de estados) deben contar con un caso de prueba documentado.
2. **Definición de Escenarios**: Cada caso de prueba debe especificar precondiciones, datos de prueba (inputs), pasos a seguir y resultado esperado.
3. **Escenarios de Borde**: Validar activamente campos vacíos, entradas duplicadas, códigos inexistentes y transiciones inválidas.
4. **Resguardo de Evidencias**: Las capturas de pantalla, logs u otras pruebas de ejecución deben almacenarse de forma organizada en la carpeta `deliverables/evidencia/`.
5. **Reportes**: Cada ciclo de auditoría de calidad debe generar un reporte tabular del estado de los casos de prueba.
